# まだやらなくていい

締め切りギリギリまで着手しない人のための、先延ばし支援サービスです。

ユーザーがタスク名と締め切りを入力すると、締め切り、作業量、過去の傾向から「まだ始めなくてよい時間」を推定します。あわせて、成功確率、リスク帯、サボってよい時間、その時間に合った娯楽や現実逃避プラン、間に合わなかった場合の言い訳文を提示します。

このサービスが人をダメにするのは、作業を代行するからではありません。ユーザーから「今やるべきか」を判断する力を奪い、見積もり、自制、反省、言い訳までをアプリに委ねさせるからです。

普通のタスク管理アプリは、早く始めること、効率よく終わらせること、忘れずに行動することを支援します。このサービスは逆に、ユーザーが罪悪感を減らしながらギリギリまでサボれるようにします。

また、多くの代行系AIは、調べる、整理する、作業を進めるといった「やること」を肩代わりします。しかし、それは人をより効率的に働かせる支援です。本当に人をダメにするのは、作業そのものの代行ではなく、行動するかどうかの判断や、失敗から学ぶ責任感そのものをAIに預けることです。

## なぜ人をダメにするのか

このサービスが人をダメにする理由は、作業を代行するからではありません。  
もっと根本的に、ユーザーから **自分で始める判断** を奪うからです。

ユーザーは、本来なら自分で考えるべきことをアプリに委ねます。

- このタスクはどれくらい重いのか
- 今始めるべきなのか
- まだサボっても大丈夫なのか
- 危険になったらどう逃げるのか
- 間に合わなかったら何と言い訳するのか

つまり、このサービスは単なる便利ツールではなく、**自制心、見積もり、反省、言い訳までを外注するサービス**です。

## 中核体験

ユーザーがタスク名と締め切りを入力すると、サービスは以下を提示します。

- 最も遅い着手時刻
- 成功確率
- 安全圏、危険圏、詰みかけのリスク帯
- 今サボってよい時間
- その時間に合った休憩、娯楽、現実逃避プラン
- 間に合わなかった場合の言い訳文

推定は断定ではなく、成功確率やリスクとして扱います。  
そのため、悪ふざけだけではなく、「先延ばしを前提にした意思決定支援」として成立します。

## このサービスの危険な価値

このサービスは、ユーザーを無理に更生させません。  
むしろ、先延ばしする人間の弱さを前提にして、その弱さを気持ちよく使える形に整えます。

やる気を出させるのではなく、まだやらなくていい理由を与える。  
反省させるのではなく、次に逃げるための履歴を残す。  
失敗を責めるのではなく、言い訳の下書きを用意する。

この徹底した方向性によって、「人をダメにするサービス」というテーマを、説明だけでなく実際に操作できる体験として表現します。
